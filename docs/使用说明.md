# 多作者博客模板使用说明

## 1. 项目简介

这是一个纯静态博客模板，支持：

- 多作者文章（单作者或多作者）
- 分类页、标签页、作者页、搜索页
- 首页最新文章与标签云
- 文章封面与页面背景图
- 文章隐藏与密码保护
- Giscus 评论系统
- 一键构建为 `dist/` 静态产物

适用于不想引入复杂后端、希望低成本部署博客的场景。

## 2. 环境要求

- Node.js 18+（推荐 LTS）
- npm 9+
- 可选：Python 3（用于本地静态预览服务器）

## 3. 安装与启动

在项目根目录执行：

```bash
npm install
npm run build
npm run dev
```

打开 `http://localhost:4173` 预览站点。

## 4. 项目结构

```text
content/posts/          # 文章 Markdown
content/authors/        # 作者 Markdown
content/images/         # 封面图、背景图等资源
src/templates/          # 页面模板
src/assets/             # 样式
src/static/             # 静态配置（如 comments.json）
scripts/build.mjs       # 构建脚本
dist/                   # 构建输出目录（部署目录）
```

## 5. 发布文章

在 `content/posts/` 新建一个 Markdown 文件（例如 `my-first-post.md`）：

```md
---
title: 我的第一篇文章
date: 2026-04-13
author: 作者A
# 或多作者：
# authors: 作者A, 作者B
category: 技术
tags: JavaScript, 静态博客
cover: /images/cover-hello.svg
background: /images/cover-build.svg
hidden: false
locked: false
# password: your-password
summary: 一句话摘要（可选）
---

这里写文章正文。
```

字段说明：

- `author` / `authors`：二选一，`authors` 支持多个作者
- `category`：用于分类页聚合
- `tags`：用于标签云和标签详情页
- `hidden: true`：从首页/列表隐藏，只保留直链访问
- `locked: true` + `password`：启用密码查看正文
- `cover`：文章卡片封面
- `background`：文章页面背景图

## 6. 新增作者

在 `content/authors/` 新建 Markdown（例如 `author-a.md`）：

```md
---
name: 作者A
headline: 全栈开发者
summary: 关注前端工程化与内容创作
background: /images/cover-build.svg
---

这里是作者简介正文。
```

构建后自动生成：

- `/authors/` 作者列表页
- `/authors/<slug>/` 作者详情页和该作者文章列表

## 7. 评论系统（Giscus）

配置文件：`src/static/comments.json`

至少填写以下字段：

- `repo`
- `repoId`
- `category`
- `categoryId`

并设置：

```json
{
  "enabled": true
}
```

常用参数：

- `mapping`：文章与评论映射方式（默认 `pathname`）
- `lang`：评论语言（默认 `zh-CN`）
- `themeLight` / `themeDark`：明暗主题样式

## 8. 本地开发流程建议

1. 新增或修改 `content/posts/*.md`
2. 运行 `npm run build`
3. 运行 `npm run dev` 预览
4. 检查分类、标签、作者页是否正常聚合
5. 提交到 Git

## 9. 部署说明（Cloudflare Pages）

1. 将仓库推送到 GitHub
2. 在 Cloudflare Pages 连接该仓库
3. 构建配置填写：
   - Build command: `npm run build`
   - Build output directory: `dist`
4. 部署完成后访问 Cloudflare 分配域名

## 10. 常见问题

- 构建后页面空白：确认是否有 Front Matter 格式错误（冒号、缩进、日期格式）
- 文章不显示：检查是否设置了 `hidden: true`
- 密码文章不可见：确认 `locked: true` 且 `password` 不为空
- 评论不出现：确认 `comments.json` 的 `enabled` 与仓库 ID 配置正确

## 11. 推荐初始化步骤（新站点）

1. 删除 `content/posts/` 中示例文章（按需）
2. 新建你自己的作者资料
3. 替换 `content/images/` 下默认图片
4. 配置 `src/static/comments.json`
5. 首次构建并部署
